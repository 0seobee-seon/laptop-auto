# 선엔지니어링 노트북 지급현황 자동 최신화

## 목적

매월 최신 인사기록부와 전산기기 지급현황 파일을 기반으로 노트북 지급 마스터 파일을 자동 갱신한다.  
핵심 스크립트: `scripts/update_laptop_master.py`

---

## 입력 파일

| 파일 | 필수 | 필요 시트 |
|------|------|-----------|
| 인사기록부 (xlsx) | **필수** | `(주)선종합` (재직자), `퇴사자명부` |
| 전산기기 지급현황 (xlsx) | 선택 | 모든 시트 자동 인식 |

**지급현황 파일 지원 포맷** (자동 감지):
- **포맷 A** — `개인별 노트북 지급현황` 양식: 성명(F열)·현장명·모델명·모델번호·인수일자·비고
- **포맷 B** — `선 노트북출고현황` 양식: 모델명·S/N·날짜·성함·현장

---

## 출력

| 항목 | 경로 |
|------|------|
| 마스터 파일 | `D:\D\김영섭\현장\감리현장 전산기기 납품내역서\선엔지니어링_노트북지급현황_최신화.xlsx` |
| 변경 로그 | 같은 폴더의 `update_log.json` |

**마스터 파일 시트 구조:**
- `노트북보유` — 노트북 이력 있는 재직자 (인수일자 오름차순)
- `미지급`     — 노트북 이력 없는 재직자
- `요확인_중복` — 선종합·퇴사자 양쪽에 이름이 있는 인원

---

## 실행 절차

### 1단계: 파일 경로 추출
사용자 메시지의 `@` 참조 또는 언급에서 파일 경로를 추출한다.  
인사기록부가 없으면 먼저 요청한다.

### 2단계: 스크립트 실행

```bash
python ".claude/skills/seoneng-laptop-update/scripts/update_laptop_master.py" \
  --hr "<인사기록부 경로>" \
  [--new-equip "<지급현황 경로>"] \
  [--out "<출력 경로>"]
```

- `--out` 생략 시 → 기본 마스터 경로에 덮어씀
- `--new-equip` 생략 시 → HR 정보만 갱신 (노트북 이력은 마스터에서 유지)

### 3단계: 결과 보고
스크립트가 JSON을 출력하면 아래 형식으로 사용자에게 보고한다.

## 보고 형식

```
✅ 노트북 지급현황 최신화 완료 (YYYY-MM-DD HH:MM)

**현황 요약**
| 구분 | 인원 |
|------|------|

**이번 업데이트 변경사항**
| 항목 | 인원 | 이름 |
|------|------|------|
| 신규 노트북 지급 | N명 | 홍길동, ... |
| 노트북 교체·변경 | N명 | ... |
| 미지급 → 노트북보유 이동 | N명 | ... |
| HR 정보 변경 (소속·직책) | N명 | ... |
```

## 사용 예시

**인사기록부만 제공 (HR 정보 갱신 + 노트북 이력 유지):**
```
사용자: @"인사기록부(2026.06.20).xlsx" 노트북 현황 최신화해줘
```

**인사기록부 + 신규 지급현황 함께 제공:**
```
사용자: @"인사기록부(2026.06.20).xlsx" @"개인별 노트북 지급현황(2026.06~).xlsx" 업데이트해줘
```

**다른 경로에 저장:**
```
python ... --hr "인사기록부.xlsx" --new-equip "지급현황.xlsx" --out "백업_2026-06.xlsx"
```

## 엣지 케이스 처리

| 상황 | 처리 방식 |
|------|-----------|
| 선종합·퇴사자 양쪽에 같은 이름 | `요확인_중복` 시트로 분리, 메인 시트 제외 |
| 마스터 파일 없음 (최초 실행) | 새로 생성 |
| 같은 사람에게 여러 이력 | 날짜가 가장 최근인 이력 채택 |
| 지급현황 파일의 날짜 포맷 혼재 | 자동 파싱 (25.9월 / 2023년01월 / 2013.03월 등) |
| 이름 사이 공백 차이 ("홍 길 동" vs "홍길동") | 공백 제거 후 매칭 |
